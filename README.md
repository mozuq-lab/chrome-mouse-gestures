# Mouse Gestures（Chrome用マウスジェスチャ拡張機能）

右ボタンドラッグでページ操作を行う、個人利用向けの Chrome 拡張機能です。
Manifest V3 / バニラ JavaScript で実装しており、ビルドステップは不要です。

## ジェスチャ一覧

矢印はマウスを動かす方向を表します（→右 ←左 ↑上 ↓下）。
右ボタンを押したままドラッグし、ボタンを離すと動作が実行されます。

| ジェスチャ | 動作 |
| ---------- | ---- |
| →↑ | ページ最下部へスクロール |
| →↓ | ページ最上部へスクロール |
| ↑↓ | タブを更新（リロード） |
| ←↑ | タブを閉じる |
| ←↓ | 最後に閉じたタブを復元する |
| →  | 1つ前の履歴に戻る（戻る） |
| ←  | 1つ先の履歴に進む（進む） |

> 補足: 「→で戻る／←で進む」はスマートフォンのスワイプ操作のイメージに
> 合わせたもので、方向の取り違えではありません。

ジェスチャ定義は `content.js` 冒頭の `GESTURES` オブジェクトにまとまっています。
ここを編集するだけで割り当てを変更・追加できます。

## インストール手順（パッケージ化されていない拡張機能を読み込む）

1. このリポジトリをローカルに用意します（クローンまたはダウンロード）。
2. Chrome で `chrome://extensions` を開きます。
3. 画面右上の **デベロッパーモード** を **ON** にします。
4. **「パッケージ化されていない拡張機能を読み込む」** をクリックします。
5. この拡張機能のフォルダ（`manifest.json` がある階層）を選択します。
6. 一覧に「Mouse Gestures」が表示されれば完了です。

### 注意事項

- **デベロッパーモードは ON のまま維持してください。**
  パッケージ化されていない拡張機能は、デベロッパーモードを OFF にすると
  無効化されます。
- **一部のページではジェスチャが反応しません。**
  `chrome://` 系のページ、新しいタブページ、Chrome ウェブストアなどでは
  Chrome の仕様上 content script が動作しないため、ジェスチャは無効です
  （エラーは出さず単に無視されます）。
- **macOS では右クリックのコンテキストメニューが表示されなくなります。**
  macOS の Chrome は右ボタンを「押した瞬間」にコンテキストメニューを開く仕様で、
  メニューが開くとそれ以降のマウス操作がページに届かず、ジェスチャを認識できません。
  そのため macOS ではジェスチャを成立させるために、右ボタン操作中のメニュー表示を
  常に抑制しています（押した時点ではジェスチャか単なる右クリックか区別できないため）。
  Windows / Linux ではこの制約はなく、移動を伴うジェスチャのときだけメニューを抑制し、
  単なる右クリックでは通常どおりメニューを表示します。
  - **Shift + 右クリック**ではジェスチャを開始せず、コンテキストメニューを表示します。
    macOS でメニューを出したいときのエスケープハッチとして利用できます。

## 仕組み（概要）

- **ジェスチャ判定（`content.js`）**
  - 右ボタン押下（`mousedown` / `e.button === 2`）でトラッキングを開始。
  - `mousemove` で移動を追跡し、起点から約 30px を超えたら主軸（|dx| と |dy| の
    大きい方）と符号から方向（U/D/L/R）を確定。
  - 直前と異なる方向に変わったときだけシーケンスに追加（連続する同方向はまとめる）。
  - `mouseup` で確定したシーケンスを `GESTURES` と照合して動作を実行。
  - ジェスチャが成立した（移動があった）場合のみ、直後の `contextmenu` を
    キャンセル。移動のない単なる右クリックは通常どおりメニューを表示します。
- **動作の実行**
  - ページ内で完結する操作（スクロール・戻る／進む）は content script 内で実行。
  - タブ単位の操作（リロード・閉じる・復元）は `chrome.runtime.sendMessage` で
    service worker（`background.js`）へ依頼し、`chrome.tabs` / `chrome.sessions`
    API で実行します。
- **軌跡表示**
  - ジェスチャ中はビューポート全体を覆う透過 `<canvas>` を生成してカーソルの
    軌跡を線で描画し、`mouseup` で消去します（`pointer-events: none` のため
    ページ操作の邪魔にはなりません）。

## ファイル構成

| ファイル | 役割 |
| -------- | ---- |
| `manifest.json` | 拡張機能の定義（Manifest V3） |
| `background.js` | service worker。タブ操作を担当 |
| `content.js` | ジェスチャ判定・軌跡描画・ページ内操作 |
| `content.css` | 軌跡 canvas のスタイル保険 |

## 権限について

- `permissions`: `tabs`（タブのリロード／クローズ）, `sessions`（閉じたタブの復元）
- `host_permissions`: `<all_urls>`（全ページで content script を動作させるため）

個人利用を想定しており、外部への通信は一切行いません。
